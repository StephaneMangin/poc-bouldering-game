# Playtest Plan

## Objective
Validate readability, fun, and difficulty of the MVP before freeze.

## Tester Profiles
1. Group A: 6 players new to technical games.
2. Group B: 6 players experienced with precision/challenge games.

## Test Scenarios
1. Tutorial + first run on R1 in Arcade.
2. R3 run on Medium.
3. Keyboard remapping via interface.
4. Simulation attempt on R4 or R5.

## Data to Collect
1. Completion rate per route/mode.
2. Average run time.
3. Number of falls and associated causes.
4. Chalk/focus usage.
5. Number of successful keyboard rebindings.

## Post-Test Questions
1. Did you understand why you fell?
2. Did the visual/audio feedback help you?
3. Did the control feel right?
4. Do you want to immediately start another run?

## Validation Thresholds
1. Controls comprehension >= 80%.
2. Failure cause readability >= 70%.
3. Intent to replay >= 70%.
