# Sprint Cadence Checklist

## Before Sprint (planning)
1. Sprint scope confirmed (max 1 main objective).
2. Definition of Done validated.
3. Top 3 risks identified + mitigation.
4. Target KPIs explicit.
5. External dependencies verified.
6. Tasks broken down into testable increments.

## Mid-Sprint (monitoring)
1. Progress review at D2/D3.
2. If drift > 20%, reduce secondary scope.
3. Re-test critical flows after each fix.
4. Update tuning journal if values changed.

## End of Sprint (review)
1. Verify each DoD item.
2. Document gaps: done / not done / deferred.
3. Capture lessons learned (max 3 points).
4. Update `plan_sprint_traceability.md`.
5. List prerequisites for the next sprint.

## Definition of Deferral
An item is deferred if:
1. The validation criterion is not achievable in <= 1 remaining day.
2. It blocks a P0 item of the sprint.
